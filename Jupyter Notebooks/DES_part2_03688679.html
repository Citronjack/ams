2.8.1 System with different Queue capacity¶

I see 2 trends:

  1. Variance of the Queue length decreases: This effect happs for the exact reason that I raised the simulation time! With more data points I can now more accuratly determine the real expectation and variance since the underlying distrubtuion is more well defined when I have access to more data points that are not influenced by the start up of the system (since I start with an empty buffer)
  2. Mean of the Queue length increases with increasing buffer size: It increases in the direction of 3 for S=5 and 4 for S=6 & S=7. This shows that more than 4 buffer slots are rarly needed for the chosen IAT and unifrom processing time of the packets.
  3. Variance of the Waiting time decreases with increasing buffer size: In comparison it becomes apparent that the distribtuion have become significatly peakier, reduced variance. This happens for the same reason mentioned in the last ex. and above
  4. Mean of the waiting time increases with increasing buffer size

So, to understand it more cleary, I have in a previous work poltted myself this in a figure similar to the one below. As you can see, for the longer simulation time the starting up phase has much less of an influence since the total number of packages is higher, from a ~400 to ~4000. So, the mean slightly rises. (Be carefull, when I plot the line I force same length for all simulations, so what you see is the least amount of packets in a run! i.e. it should be a bit less than expected but it still serves well for my argument)

2.8.2 Choosing the right number of bins¶

For the mean queue length histogram it is quite obvious to use S+1 bins, since 0 is possible. For me, it was much more difficult to write the code for the plot but ok.

For the lineplot I had to consider a bit more. First, I ran a few simulations to get a feeling for the maximum waiting time. I found that values above 3000ms are very rare and I decided to be save and set it to 4k ms. The number of bins is more important. At first, I have 20 bins, so one for every 200 ms, it turns out this is not enough, especially the plot with longer runtime becomes much to peaky at a single value. SO I doubled it to 40, I realized there are now to many spikes, like a really pointy mountain has. So, I lowered it to 30. THis looks good now.

Funily enough, I didnt really the rule of thumb to be much of use, I always have to consider the actual data I want to represent

2.8.3 Variance Calculation¶

If I had an infinite amount of data samples, the variance is normalized with 1/n. For a inite number of smples and also in my code I normalize the matrix by 1/(n-1). The difference is explained in the lecture as shown on the picture below. We see that the relation between the variance of X and the variance of the estimator of X is defined by a facotr 1/n. I will now try to explain where this comes from.

  1. We always operate on a sample of all possible outcomes of our experiment that we want to know the variance of. So, directly calculating the mean and variance ^2 (formula with 1/n) means that they are random variables two, since it is possible to choose different samples to calculate the variance and mean from.
  2. If we now do a funny thing and calculate the mean of the variance ^2 we find that it is biased as E[^2] = (n-1)/n*^2
  3. If we want to get rid of that n-1 we simply divide by it and voil there is the unbiased emperical variance formula
