3.3.1 System with different Queue capacity¶

To understand this we first have to understand how the radom number generators work. When given the same seed twice and genearte a random sequence using a random number generator such as the one in the lib random, we see that the two sequences are exatly the same! Not so random after ha - this makes sense since we can better understand how a change in the system design affects performance, otherwise we could never be certain if it was just by acciedent that the system performed differently (or not). THus the results of simulations are deterministic and can be compared.

import random as r

def get_random_seq(a):
  rl = []
  for i in range(a):
    rl.append(r.uniform(0,1)) 
  return rl

r.seed(47)
r1 = get_random_seq(5)
r.seed(47)
r2 = get_random_seq(5)
print(r1, "\n", r2)

3.3.2 Choosing the right number of bins¶

I use n=10k realization of my 2 distribtuions for the plots above. I decided to use the rule of thumb that says to use sqrt(n) to plot the histograms. As we can see, this seems to be a good match since we can quite clearly figure out that the underlying distribution is a exponential or unifrom one. If we raise the number of bins to 1k, the histograms become much peakier. If we lower the amout to just 10 the histograms beocme flatter. We can still from both figure out what the underlying distribtuion is but also notice that the number of bins is either too high or too low (check the empirical rate).

Thus the rule of thumb seems to be a very good pick for these plots.

6.6.6/2.2.2 = 3.3.3 System utilization¶

AS can be seen in the figure above, the utiliation increases with increasing rho. This makes sense since rho gives the theoretical utiliuation. In fact, the utilization and rho are almost equal for until rho = 0.9, then the utilizatioon is a bit lower. The difference stems from packet loss, since the packet is lost it will not be processed and thus the server is not utilized.

Increasing the simulation time to 1000s slightly increases the utilization, I think this trend continues and for a infinite simulation time we would reach the rho.

Increasing the buffer size to inf would eliminate packet loss, thus the utiliation should be closer to rho. This is true, see figure below.
